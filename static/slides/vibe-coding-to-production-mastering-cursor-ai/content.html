How ChatGPT “talks”: tokens 101

  • Models don’t “see characters” — they process tokens (subword units).
  • Tokenization is usually BPE-style: frequent chunks get their own tokens.
  • Generation = predicting the next token, one at a time, with a probability distribution.
  • Why you care: context window size, cost, speed, and accuracy.

Split → Encode → LLM “thinking” → Output → Decode → Join

  1. Split text into tokens via tokenizer.
  2. Encode tokens → ids (integers).
  3. LLM “thinking”: attention layers transform vectors, predict next token id.
  4. Output stream of token ids.
  5. Decode ids → tokens; Join tokens → text.

Demo this visually; tie back to cost & window budgeting.

Hands-on token view (Python + tiktoken)

import tiktoken
enc = tiktoken.get_encoding("cl100k_base")
text = "Hello, world! 👋 Ship fast."
ids = enc.encode(text)
print("ids:", ids)
print("tokens:", [enc.decode([i]) for i in ids])
print("count:", len(ids))

Context windows & budgeting

  • 8K ≈ a few files; 32–200K ≈ small repos.
  • Feed only what the agent needs now.
  • Pin a 1-pager brief (.claude.md) to avoid drift.

Task prompt (backend)

Implement /api/projects and /api/tasks:
- Go 1.22, chi router, SQLite via modernc.org/sqlite
- Migrations: projects(id PK, name, created_at), tasks(id PK, project_id FK, title, done, created_at)
- Handlers: list/create/update/delete
- Return JSON; validate input; use prepared statements
- Add unit tests covering happy path + 1 error case

Agent prompts — React frontend

Create a Vite + React app with:
- Pages: Projects (list/create), ProjectDetail (tasks CRUD), Notes panel
- Data via fetch to /api/*
- Accessible forms (labels/roles), optimistic UI on task toggle
- Routing with react-router; basic error boundary
- Add Playwright test: create project, add task, mark done, assert UI

Keep components small; name by role; test selectors by role.

API endpoints & schema

GET  /api/projects/:id/tasks

-- migrations/001_init.sql
CREATE TABLE projects (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE TABLE tasks (
  id INTEGER PRIMARY KEY,
  project_id INTEGER NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  done INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
